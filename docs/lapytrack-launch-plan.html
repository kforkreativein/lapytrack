Path 1

Fastest Launch

Use this when you want the current app online quickly without rewriting the backend.

Frontend
Vercel deploys krish-computer-app/frontend.
Backend
Fly.io or Render runs the existing FastAPI Docker app.
Database
MongoDB Atlas works with the current MongoDB code.
Storage
Start with Fly volume; move to R2/S3/Supabase Storage for SaaS.

Frontend Environment

REACT_APP_BACKEND_URL=https://lapytrack-api.fly.dev

Do not include /api. The app code appends /api itself.

Backend Environment

MONGO_URL=mongodb+srv://USERNAME:PASSWORD@cluster.mongodb.net/?retryWrites=true&w=majority
DB_NAME=krish_computer
JWT_SECRET=long-random-production-secret
FRONTEND_URL=https://your-frontend.vercel.app
CORS_ORIGINS=https://your-frontend.vercel.app

Fly.io Deploy Commands

cd krish-computer-app/backend
fly secrets set MONGO_URL="mongodb+srv://..."
fly secrets set DB_NAME="krish_computer"
fly secrets set JWT_SECRET="long-random-production-secret"
fly secrets set FRONTEND_URL="https://your-frontend.vercel.app"
fly secrets set CORS_ORIGINS="https://your-frontend.vercel.app"
fly deploy
Path 2

Long-Term SaaS

Use this when the app needs multiple shops, subscriptions, isolated data, and durable file storage.

Frontend
Vercel, Netlify, or Cloudflare Pages.
Auth
Supabase Auth with shop membership and roles.
Database
Supabase Postgres with Row Level Security.
Storage
Supabase Storage or Cloudflare R2 for uploaded photos.
Billing
Stripe subscriptions by shop.

Required SaaS Tables

  • shops
  • users
  • shop_members
  • devices
  • movements
  • customers
  • transactions
  • files
  • subscriptions

Every business record should include shop_id. That is the core of SaaS data isolation.

Migration Note

Supabase can power the SaaS version, but the current app is FastAPI + MongoDB. Moving fully to Supabase requires a planned data-layer migration.

Launch Checklist

  • Push code to GitHub.
  • Create MongoDB Atlas cluster.
  • Deploy backend to Fly.io or Render.
  • Set backend env vars and explicit CORS_ORIGINS.
  • Deploy frontend to Vercel.
  • Set REACT_APP_BACKEND_URL to backend origin only.
  • Test PIN login, inward job creation, QR job sheet, Khata Book, reports, and photo uploads.
  • Confirm uploaded files survive backend restart.
